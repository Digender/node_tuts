Package versioning explained
  
  major.minor.patch
  * Major - Some major change that might effect our code because something drastic has changed in library.
  * Minor - Minor means a small change that has been added or a code is deprecated. This will not have any effect on our current library used
  * Patch - Small patch to fix bug or small amount of code that won't impact much
  * ^ - this says npm to go and update minor and patch versions
  * ~ - Only patch updates
